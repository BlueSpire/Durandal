mapUnknownRoutes

(
  • [config]
)
chainable

Inherited from Router: src\plugins\js\router.js:631

Configures how the router will handle unknown routes.

Parameters:

  • [config] String | Function optional

    If not supplied, then the router will map routes to modules with the same name. If a string is supplied, it represents the module id to route all unknown routes to. Finally, if config is a function, it will be called back with the route instruction containing the route info. The function can then modify the instruction by adding a moduleId and the router will take over from there.

navigate

(
  • fragment
  • options
)
Boolean

Inherited from Router: src\plugins\js\router.js:498

Save a fragment into the hash history, or replace the URL state if the 'replace' option is passed. You are responsible for properly URL-encoding the fragment in advance. The options object can contain trigger: true if you wish to have the route callback be fired (not usually desirable), or replace: true, if you wish to modify the current URL without adding an entry to the history.

Parameters:

  • fragment String

    The url fragment to navigate to.

  • options Object | Boolean

    An options object with optional trigger and replace flags. You can also pass a boolean directly to set the trigger option.

Returns:

Boolean: Returns true/false from loading the url.
