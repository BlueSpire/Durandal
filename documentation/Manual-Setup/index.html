Manual Setup

Durandal works with any backend technology...or no backend technology. All you need to get going are the necessary script libraries, modules and folder structure. If one of the other methods of getting started does not suit you, you can always just copy the files into your own project. Your basic setup would normally be as follows:

  • App
    • durandal
      • amd
      • plugins
      • transitions
      • widgets
      • ...durandal modules here...
    • ...your app code here...

You can organize your own code in any way you see fit. Then, you can teach the framework about your structure by overriding various functions that tell it where or how to find certain components. You can even relocate plugins, widgets and transitions if you like. Just be sure to configure the framework to locate those components.

Note: You may wish to use the Mimosa skeleton as a starting point. It's located at skeletons\mimosa Just remove the mimosa configuration file and you should be able to use it as is. You could also grab the contents of the repo's main App folder and it's index.html and start with that.
