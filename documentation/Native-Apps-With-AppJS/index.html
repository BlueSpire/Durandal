Native Apps with AppJS

Key Points
  • Optimize your app.
  • Configure AppJS to serve files from your content folder.
  • Copy all assets to the content folder of your AppJS project.
  • Update the index.html file to point to the optimized build (and your other assets).

Explanation

Creating native applications for desktop is easy by combining Durandal with AppJS. The first step is to optimize your application for deploy. How you do this depends on what platform and setup you are running. In general, you have these options:

  • If you are using the .NET project, you can run optimizer.exe, located at app/durandal/amd/optimizer.exe
  • If you are using Mimosa, then you can execute the command mimosa build -mo
  • For all other configurations, you should see the r.js documentation for help using the optimizer directly.

Ultimately, the output of this process should be a file such as main-built.js which contains all your app's JavaScript and HTML. Next, you will want to make sure that AppJS is configured to serve files from a particular folder. To do that, locate the app.js file in your AppJS project and make sure it has something like this:

app.serveFilesFrom(__dirname + '/content');

Then, copy all your applications's assets to that folder. This includes, images, css, 3rd party JS libraries...and of course, your main-built.js file. Finally, you need to update the index.html file so that it points to your css, etc and be sure to add a script reference for main-built.js. That's it.

Note: AppJS is still in the early stages of development. The integration described above may change in the future, or there may be more/better options as well. Be sure to check the AppJS documentation.
